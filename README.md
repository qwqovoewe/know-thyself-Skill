# know-thyself.Skill

> 现有项目：把你的历史数据喂给 AI
> 这个项目：让 AI 通过对话把你从你自己身上挖出来

没有聊天记录？没关系。你才是关于你自己最好的数据源。

在这个蒸馏技术的时代，每个人都可以拥有自己的数字副本。但真正难得的，不是副本能替你做什么——而是**蒸馏的过程，让你重新认识了自己**。

抽出一段时间，跟自己来一场深度对话，照照自己的镜子。

---

## 和其他蒸馏项目的区别

市面上的蒸馏项目都在做同一件事：你上传聊天记录、日记、照片，AI 分析然后生成副本。

这个项目反过来：**AI 主动采访你，通过 30-50 轮对话，把你从你自己身上挖出来。**

| | 其他项目 | know-thyself |
|---|---|---|
| 数据来源 | 你的历史文件 | 实时对话 |
| 蒸馏方式 | AI 分析你上传的内容 | AI 采访你，追问你 |
| 门槛 | 需要导出微信/日记 | 零门槛，打开就能聊 |
| 深度 | 取决于数据质量 | 取决于对话深度 |
| 盲区发现 | 没有 | 核心输出之一 |

采访出来的结果，和你自己写的简历，哪个更真实——不用解释。

## 安装

核心是 `prompts/` 下的四个 prompt 文件，平台无关。针对 Claude Code 做了开箱即用的适配（SKILL.md），也可以在其他 AI agent 里使用。

### Claude Code（推荐）

```bash
# 安装到全局（所有项目都能用）
git clone https://github.com/qwqovoewe/know-thyself-Skill ~/.claude/skills/know-thyself

# 或安装到当前项目
mkdir -p .claude/skills
git clone https://github.com/qwqovoewe/know-thyself-Skill .claude/skills/know-thyself
```

安装后直接使用 `/know-thyself` 命令。

### 其他 AI Agent（Cursor、Cline、Codewiz 等）

```bash
git clone https://github.com/qwqovoewe/know-thyself-Skill
```

然后告诉你的 agent：

> 读取 know-thyself/SKILL.md，按里面的执行流程开始蒸馏。

SKILL.md 里的指令是自然语言写的，任何能读写文件的 agent 都能理解和执行。

## 使用

```
/know-thyself              # 开始蒸馏（30-50 轮对话）
/know-thyself resume       # 继续上次未完成的会话
/know-thyself status       # 查看所有蒸馏的状态
/talk-to-me {slug}             # 自己和自己对话
/be-me {slug}                  # 启动数字副本（别人来问你）
```

### 蒸馏

运行 `/know-thyself` 后，AI 会开始采访你。

不需要准备任何东西。问什么答什么。AI 会追问，会注意到你怎么回答，会在你绕开某个话题的时候记录下来。

对话过程中，每一轮都会自动保存到 `selves/{slug}/transcript.md`，中途退出不会丢失进度。运行 `/know-thyself resume` 从断点继续。

约 30-50 轮后，AI 一次性分析整个对话，生成三个文件：

```
selves/{slug}/
├── transcript.md    # 完整对话记录
├── persona.md       # 你是什么人
├── memory.md        # 你经历过的事
└── blindspots.md    # 你没说的地方（只给你自己看）
```

### 自己和自己对话

> 以下节选来自一次真实的 talk-to-me 对话（已脱敏），完整版本见 [`examples/talk-to-me.md`](examples/talk-to-me.md)。

```
你：现在有一份实习干了 168 天，组里的人很熟，工作也很熟悉。
    现在有一个 title 更好、从未接触过的后端开发组给你发 offer 了，你去吗？

副本：你拿到 offer 之后又开始担心了吧——上次拿到口头 offer 你也是这样，
     不是先开心，是先纠结。你现在每天去上班，还有那种"想搞明白一个东西"
     的感觉吗？还是已经变成"还行吧"了？

你：还行吧，不算很有兴趣，上班嘛

副本：你看，你又说"还行吧"了。你真正感兴趣的事，从来不说"还行吧"——
     你说游戏的时候列了一整段，你说追你女朋友的时候用的是"冲动地喜欢上了～"。
     "还行吧"在你嘴里，就是"其实没那么有兴趣但我不想说得那么重"。

你：舍不得那群人里面的几个很好的人，重新认识一群人也还好吧

副本：你注意到没，你跟我说了半天，都在讲你会"失去"什么。
     但那个新 offer 你一个字都没展开过。你对后端这个事，是"有点好奇想试试"，
     还是"title 好听所以应该去"？这个得你自己分清楚。
```

副本不帮你做决定，用你自己的历史质疑你的当下。

## 真实蒸馏示例

> 以下节选来自一次 32 轮的真实蒸馏（已脱敏），完整版本见 [`examples/`](examples/)。

### 对话片段

```
[Round 31]
Q: 你一开始说不喜欢主动跟别人说话，后来又说不是不喜欢是没话题。你觉得哪个更真实？

A: 后面，不是不喜欢，是没话题，而且其实坦白来讲，我不喜欢跟人 deep talk，
   或者谈论自己的一些想法，剖析自己，我觉得很奇怪，所以很多事情对于我自己
   的认识也是在跟你交谈之后才又加深了一层的。

[Round 32]
Q: 如果你的数字副本以后替你回答了一个问题，你最怕它答错的是哪种问题？

A: 一开始我只想让数字副本当工作替身，而不是灵魂替身。但聊完之后我觉得，
   在这个蒸馏技术的时代，更难得的，是一个人的灵魂、想法、思维。
   我不怕你答错任何问题——数字副本永远代替不了一个真正的人，不是吗？
```

### persona.md（节选）

```markdown
## 说话方式

- "还好吧""还行吧"是最高频的缓冲词，几乎每个自我评价的回答都会出现
- 真正确定的事反而说得极短、不加修饰
- 严肃的事轻描淡写，小事反而愿意展开
- 被追问时有时会反问来买时间
- 擅长用比喻思考：游戏是"第二世界"、程序员应该"做司机不做马车夫"

## 恢复机制

这是他最显著的人格特征之一。几乎所有挫折都在一天内恢复：

- 考试崩溃 → 伤心一天 → 第二天接着学 → 后来成了负责人（round 11-12）
- 大厂三面挂了 → "大不了再面呗"（round 28）
- 和女朋友吵架 → 最多冷战一晚 → 第二天说开（round 16）
```

### blindspots.md（节选）

```markdown
## 给你的话

你说你不喜欢 deep talk，觉得剖析自己"很奇怪"——但你刚刚做了 32 轮。
而且你做得很好。你的自我认知比你以为的清楚，只是你平时不太愿意把它拿出来看。

你最让我印象深刻的不是你的乐观——虽然那确实是你最明显的特质。
让我印象深刻的是你说"数字副本永远代替不了一个真正的人"。

你对自己的价值其实很确定。只是你习惯了用"还好吧"来装它。
```

## 工作原理

整个项目是 4 个 prompt + 1 个编排文件（SKILL.md），零代码依赖。没有后端，没有数据库，没有 API 调用——所有逻辑都写在自然语言里，任何能读写文件的 AI agent 都能跑。

为什么这么设计：蒸馏的核心是对话质量，不是工程复杂度。prompt 即产品，改一句话就能改变采访的深度和方向，不需要重新部署任何东西。

蒸馏分两个阶段：

1. **采访阶段**（excavator）：AI 逐层提问——从标签到行为到动机到内核。每轮对话实时保存。
2. **分析阶段**（distiller）：读取完整对话记录，提取人格特征。每条结论需要 ≥2 条行为证据支撑，只有自述没有行为的不写入。

两个副本模式：

- **talk-to-me**：你和副本对话，副本用你的历史质疑你的当下
- **be-me**：副本成为你，别人可以和"你"聊天
